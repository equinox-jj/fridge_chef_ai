# 🍽️ FridgeChef AI

> Scan your fridge. Get AI-powered recipes. Cook smarter.

FridgeChef AI is a cross-platform mobile app built with Flutter that lets you photograph your fridge ingredients and instantly receive personalized recipe suggestions generated by AI. No more staring into the fridge wondering what to cook.

---

## ✨ Features

- **Fridge Scan** — Point your camera at fridge contents; the app identifies ingredients automatically
- **AI Recipe Generation** — Firebase AI (Gemini) crafts recipes from whatever ingredients are detected
- **Authentication** — Secure sign-in and sign-up powered by Supabase
- **Onboarding** — Smooth first-time user experience
- **Dashboard** — Central hub for quick access to scans and recent recipes
- **Profile** — Manage account preferences and history
- **Offline-ready** — Local database via Drift keeps data accessible without a connection

---

## 🏗️ Architecture

Feature-first **modular monorepo** managed by [Melos](https://melos.invertase.io/). Each feature is an isolated Dart package with its own `pubspec.yaml`.

```
fridge_chef_ai/
├── lib/                  # App entry point, router, DI bootstrap
├── core/                 # Design system, shared widgets, theme, assets
├── dependencies/         # Centralized dependency declarations (workspace)
└── features/
    ├── auth/             # Login, register, forgot password
    ├── onboarding/       # First-run flow
    ├── dashboard/        # Home screen
    ├── fridge_scan/      # Camera capture + ingredient detection
    ├── recipes/          # AI recipe generation + display
    └── profile/          # User settings & account
```

**State management:** BLoC pattern (`flutter_bloc`) throughout all feature packages.

**Dependency injection:** `get_it` service locator, bootstrapped in `lib/injector.dart`.

**Navigation:** `go_router` with type-safe routes via `go_router_builder` code generation.

**Functional error handling:** `fpdart` (`Either`/`Option`) for explicit, composable error paths — no unhandled exceptions.

## 🚀 Getting Started

### Prerequisites

- Flutter `>=3.x` with Dart `>=3.12.1`
- [Melos](https://melos.invertase.io/) installed globally: `dart pub global activate melos`
- A Supabase project with auth configured
- A Firebase project with AI (Gemini) and Crashlytics enabled

### Setup

```bash
# 1. Clone the repo
git clone <repo-url>
cd fridge_chef_ai

# 2. Copy environment file and fill in your keys
cp .env.example .env

# 3. Bootstrap all packages (installs dependencies across the monorepo)
melos bootstrap

# 4. Generate code (freezed, drift, go_router)
melos run generate

# 5. Run the app
flutter run
```

### Useful Melos Scripts

```bash
melos run format    # Format and auto-fix all packages
melos run generate  # Run build_runner across all packages
```

---

## 📁 Environment Variables

Create a `.env` file at the project root:

```env
SUPABASE_URL=your_supabase_url
SUPABASE_ANON_KEY=your_supabase_anon_key
```

Firebase credentials are configured via `lib/firebase_options.dart` (generated by `flutterfire configure`).
